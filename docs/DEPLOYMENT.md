# Развертывание проекта

## Назначение

Документ описывает базовую подготовку Flask-проекта к размещению на бесплатном хостинге.

## Минимальные требования

- Python 3.12+
- доступ к установке зависимостей из `requirements.txt`
- поддержка запуска WSGI-приложения

## Подготовка проекта

1. Клонировать репозиторий проекта.
2. Создать виртуальное окружение.
3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Инициализировать базу данных:

```bash
flask --app app init-db
```

5. Заполнить стартовые данные:

```bash
flask --app app seed-users
flask --app app seed-clients
```

## Локальный запуск

```bash
python app.py
```

## Переменные окружения

- `SECRET_KEY` — секретный ключ Flask
- `DATABASE_URL` — строка подключения к SQLite или другой SQL-совместимой базе
- `FLASK_RUN_HOST` — адрес запуска
- `FLASK_RUN_PORT` — порт запуска

## WSGI-точка входа

Для WSGI-хостинга используется файл `wsgi.py` и объект `app`.

## Размещение на бесплатном хостинге PythonAnywhere

1. Зарегистрировать бесплатный аккаунт на PythonAnywhere.
2. Создать Bash Console и перейти в каталог проекта.
3. Создать виртуальное окружение и установить зависимости:

```bash
mkvirtualenv bitrix24-resource --python=/usr/bin/python3.12
pip install -r requirements.txt
```

4. Инициализировать базу и заполнить стартовые данные:

```bash
flask --app app init-db
flask --app app seed-users
flask --app app seed-clients
```

5. Создать новое Web App типа `Manual configuration` на Python 3.12.
6. В настройках Web App указать виртуальное окружение.
7. Открыть WSGI configuration file и заменить содержимое на импорт объекта из `wsgi.py`.
8. Настроить путь к каталогу проекта и перезапустить приложение.

## Что нужно учесть перед публикацией

- отключить debug-режим для боевого запуска
- задать безопасный `SECRET_KEY`
- обеспечить сохранность файла базы данных
- проверить доступность статических файлов
- проверить работу авторизации, админки, аналитики и интеграции Bitrix24

## Ограничение

Фактическая публикация на бесплатном хостинге требует аккаунт пользователя на выбранной платформе, поэтому из локальной среды проекта подготовлены конфигурация и инструкция, но не выполняется вход в чужой аккаунт автоматически.
