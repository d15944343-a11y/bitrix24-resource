# Баг-репорты

## BR-001. Отсутствие таблицы при добавлении новой сущности без обновления схемы

- Приоритет: высокий
- Серьезность: S2 Critical
- Описание: при добавлении новой сущности в модель и запуске приложения на старой базе SQLite возможна ошибка из-за отсутствующей таблицы.
- Предусловия: существует ранее созданная база данных проекта.
- Шаги:
  1. Добавить в проект новую модель SQLAlchemy.
  2. Запустить приложение без пересоздания схемы.
  3. Открыть раздел, использующий новую сущность.
- Ожидаемый результат: таблица доступна, раздел открывается корректно.
- Фактический результат: приложение может вернуть внутреннюю ошибку сервера.
- Статус: исправлено

## BR-002. Возможность самоудаления администратора

- Приоритет: высокий
- Серьезность: S2 Critical
- Описание: администратор мог удалить собственную учетную запись через карточку пользователя.
- Предусловия: пользователь вошел в систему под администратором.
- Шаги:
  1. Открыть карточку собственной учетной записи.
  2. Нажать кнопку удаления пользователя.
- Ожидаемый результат: удаление собственной учетной записи запрещено.
- Фактический результат: учетная запись администратора могла быть удалена.
- Статус: исправлено

## BR-003. Некорректная обработка некорректного email в формах

- Приоритет: средний
- Серьезность: S3 Major
- Описание: формы могли принимать некорректный email без явной серверной проверки.
- Предусловия: открыта форма входа, клиента, пользователя или обратной связи.
- Шаги:
  1. Ввести email в некорректном формате.
  2. Отправить форму.
- Ожидаемый результат: отображается сообщение об ошибке, форма не сохраняется.
- Фактический результат: некорректное значение могло быть принято.
- Статус: исправлено
