# Тест-план

## Идентификатор тест-плана

`TP-B24-001`

## Введение

Документ описывает подход к тестированию веб-приложения по теме:
`Разработка информационного ресурса для интеллектуального анализа клиентской базы на основе CRM системы Битрикс 24`.

## Объект тестирования

- Flask-приложение управления клиентской базой
- модуль авторизации и ролей
- личный кабинет
- панель администратора
- клиентская база
- форма обратной связи
- аналитика, сегментация, рекомендации, отчеты
- интеграция с Bitrix24 через webhook

## Функции, которые будут протестированы

- запуск приложения и доступность основных страниц
- авторизация пользователей
- разграничение прав доступа по ролям
- управление пользователями в админ-панели
- создание, редактирование, удаление и просмотр клиентов
- поиск и фильтрация клиентов
- работа формы обратной связи
- просмотр и обработка обращений
- сохранение настроек интеграции Bitrix24
- проверка подключения и импорт клиентов из Bitrix24
- аналитические и отчетные страницы
- экспорт отчета в CSV

## Функции, которые не будут протестированы

- нагрузочные сценарии промышленного уровня
- интеграция с внешними сервисами, кроме Bitrix24
- мобильные приложения, так как проект реализуется как веб-приложение

## Подход к тестированию

Для проекта планируется использовать не менее 4 методов тестирования:

- позитивное тестирование
- негативное тестирование
- smoke-тестирование
- функциональное тестирование
- интеграционное тестирование
- ручное тестирование
- автоматическое тестирование

## Критерии начала тестирования

- приложение запускается без критических ошибок
- тестовая база данных создана
- демонстрационные пользователи и клиенты добавлены
- основные страницы доступны

## Критерии окончания тестирования

- выполнены все запланированные тест-кейсы
- критические и блокирующие дефекты устранены
- результаты тестирования зафиксированы
- подготовлены баг-репорты по найденным дефектам

## Результаты тестирования

- заполненные тест-кейсы
- протокол прохождения smoke-тестов
- список найденных дефектов
- итоговое заключение о готовности системы

## Задачи тестирования

- проверить корректность работы пользовательских сценариев
- проверить обработку некорректных данных
- проверить доступ к разделам в зависимости от роли
- проверить корректность работы CRUD-операций
- проверить стабильность ключевых маршрутов

## Ресурсы системы

- ОС Windows
- Python 3.12+
- Flask
- SQLite
- современный браузер: Chrome, Firefox, Edge, Яндекс.Браузер

## Роли и ответственность

- разработчик: исправление ошибок, поддержка тестовой среды
- тестировщик/студент: подготовка тест-кейсов, ручное выполнение сценариев, фиксация результатов
- руководитель проекта: согласование результатов тестирования

## Расписание

- smoke-тестирование после ключевых этапов разработки
- функциональное тестирование после реализации основных модулей
- итоговое регрессионное тестирование перед сдачей проекта

## Риски

- ошибки в ролевой модели доступа
- потеря данных при CRUD-операциях
- некорректная работа интеграции Bitrix24
- различия поведения в разных браузерах

## Согласование

Тест-план подлежит уточнению и расширению по мере развития проекта.
